Making Changes

If you are unfamilar with git and github first review Good Resources for Learning Git and GitHub and GitHub Guides.

First clone the documentation project.

Next, find one of the sections below that match what you are doing and follow the instructions.

Once your changes have been merged into upstream master they will show on the live site within a couple minutes.

Modify an Existing Document

  • Find file that you want to edit
  • Make your changes
  • Commit your chages
  • Push your changes, either to master upstream or to your fork
  • If you have pushed to your fork, send a pull request for someone to review and merge
Congratulations. You have now made a change to the documentation.

Creating new Content

  • Find a folder that matches what you want to write
  • Create a new markdown document
  • Edit _config.yml navagation section to add your new markdown file or folder
  • Commit your chages
  • Push your changes, either to master upstream or to your fork
  • If you have pushed to your fork, send a pull request for someone to review and merge
Congratulations. You have now made a change to the documentation.

Repository Structure

  • `_includes` - *special folder* contains snippets that can be included in other pages
  • `_layouts` - *special folder* contains the layouts that are shared between pages. Layouts can be inherited, the root layout is `index.html`.
  • `_site` - the output of the generated site is stored here by default, this folder only exists after Jekyll built the site
  • `_assets` - contains the main stylesheet and javascript
  • `images` - stores the images used in pages
Any other folders are most likely user created content
